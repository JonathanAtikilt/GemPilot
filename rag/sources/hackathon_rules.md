# Hackathon Rules

MVPilot should optimize for a working demonstration over broad feature coverage. Judges should be able to understand the problem, see the core workflow run, and inspect evidence that the system used relevant constraints.

The MVP must avoid hardcoded secrets, clearly document setup steps, and keep backend-only credentials out of frontend code. Any external API dependency should fail with an actionable error when credentials are missing.

For first build priority, prefer the smallest vertical slice that connects agents, project context, build feedback, and a user-visible result.
